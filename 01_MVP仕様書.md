# 養豚現場相談AI MVP 仕様書

## 1. プロジェクト目的と基本思想
養豚現場の担当者が、雑な相談や気になることをそのまま入力（または写真添付）すると、AIが「まずやること」を短く返し、必要に応じて理由を補足し、危険な場合は獣医師相談を促すMVP（最小プロトタイプ）を作成する。

**基本思想**
- **現場判断AIである:** 病名当て（診断）や治療の断定は行わない。目的は現場判断の整理、確認漏れ防止、および現場→獣医師への橋渡しである。
- **MVPの最優先:** 動作する最小構成の完了を最優先する。将来の拡張性よりも現在のシンプルさを重視する。
- **入力ハードルを下げる:** 現場の面倒な入力を避け、主ユーザー（本を読まない、細かい入力を嫌うが困ったら質問はする現場スタッフや管理者）が直感的に使えるようにする。

## 2. PO（Product Owner）によるMVPスコープ定義
MVPとして「やるべきこと」と「やらないこと」を明確化する。

### 2.1. 必須機能（MVP範囲内）
- 自由記述による相談入力（テキストテキストエリア）
- 写真添付機能（任意・1枚のみ）
- AIによる短文回答（固定フォーマット）
- 危険なケースにおける獣医師相談の促進表示
- システム内部でのJSON形式での構造化出力保持

### 2.2. 除外機能（MVP範囲外）
- 病名、治療指示の断定
- 高度な画像診断（本MVPでは画像は「現場状況の把握補助」としてのみ使用）
- ユーザー認証、マルチテナント設計、複雑な権限管理
- 農場ごとの複雑な個別設定
- ROI分析、詳細ダッシュボードの提供
- （※会話ログの永続的なデータベース保存は将来的な拡張とし、MVPでは最小限の保存あるいは揮発でも可とするが、出力JSONの形式は固定化しておく）

## 3. 入出力仕様の概要
- **入力:** 自由記述テキスト＋写真(0〜1枚)
- **出力ルール:** 定型フォーマットに従い、結論から短く返す。
  1. まずやること（1〜3個）
  2. 緊急度（すぐ獣医師相談、今日中に確認、まず様子確認 など）
  3. 理由（1〜2文程度）
  4. 獣医師への相談目安（必要時のみ）
  5. 追加で分かれば役立つ情報（任意）
