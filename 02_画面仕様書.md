# 養豚現場相談AI MVP 画面仕様書

## 1. 画面構成方針
- **デバイス:** スマートフォン前提
- **ページ構成:** 1画面で完結させる（Single Page Application的な挙動、遷移を極力減らす）
- **表示方針:** 最初に結論が目に入ること、長文を避けること、回答をカードUIとして見やすくまとめること。

## 2. UI要素とレイアウト

### 2.1. 相談入力エリア
- **テキストエリア:** 自由記述で入力可能な複数行テキストボックス。「最近下痢が多い」「咳が増えた気がする」などを想定。
- **写真添付ボタン:** 最大1枚の画像を添付できるボタン（任意入力）。アップロード済みの場合はサムネイルを表示するか、ファイル名を表示＋削除ボタンを配置。
- **送信ボタン:** 入力内容をAIに送信するボタン。

### 2.2. 状態（ステート）表示
- **未入力状態:** 送信ボタンを非活性（Disabled）にするか、タップ時に「相談内容を入力してください」とバリデーションエラーを出す。
- **ローディング:** 「AIが考えています...」等のスピナーやスケルトンUIを表示し、処理中であることを明示する。
- **エラー時:** 通信エラーやサーバーエラー時に「回答の取得に失敗しました。もう一度お試しください。」というメッセージを表示。リカバリー手段（再試行ボタン）を設ける。

### 2.3. AI回答表示エリア（カードUI）
回答は以下の固定フォーマットのカードとして表示する。

1. **まずやること:** 箇条書き（1〜3点）。太字などで最も目立たせる。
2. **緊急度:** 目立つバッジやカラーテキストで表示（例：🔴すぐ獣医師相談、🟡今日中に確認、🟢まず様子確認）。
3. **理由:** 1〜2文程度の簡潔なテキスト。
4. **獣医師への相談目安:** 該当する場合のみ表示。「注意項目」としてアイコン付きで表示。
5. **追加で分かれば役立つ情報:** 任意項目。折りたたみ（アコーディオン）UIなどにして、初期表示の文字数を抑える工夫をする（「詳しく見る」等）。
- **（将来用・任意）** 「獣医共有用メモを作る」ボタン：クリックで回答と入力をまとめたテキストをクリップボードにコピーする機能。MVP時点で余裕があれば実装。
