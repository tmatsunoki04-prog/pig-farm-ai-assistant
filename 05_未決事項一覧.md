# 養豚現場相談AI MVP 未決事項一覧 (Decision Log)

現在MVP仕様を策定した段階で、将来の拡張や実装時に決定すべき未決事項を以下にまとめる。

1. **写真付き相談時の具体的なプロンプト・インフラの扱い**
   - 状態: 未決
   - 詳細: MVPにおいて、画像つきリクエストをLLM（例えばGPT-4Vなど）に直接投げるのか、一旦画像URL化してバックエンドで処理するのか、インフラの具体的な接続方法が決まっていない。まずはダミーAPIまたはテキスト同等のモックでフロント実装を進めるか、即座にVLMへ繋ぐか、今後のArchitectタスクにて最小構成を決定する。

2. **`image_card_id` を用いた定型画像カードの表示の仕組み**
   - 状態: 未決
   - 詳細: 将来的に「下痢が増えたときにまず見る3点」などのクラウド上の画像リソースをフロントエンドで引っ張る仕組み。MVPではフロントエンド側は `image_card_id` を受け取るが無視する（あるいはダミー画像を出す）だけの対応で良いか、POと合意が必要。

3. **「獣医共有用メモを作る」ボタンの実装有無**
   - 状態: 未決
   - 詳細: ユーザー要望にあるが必須ではない。クリップボードコピー機能を実装する工数がMVPスコープに収まるのであれば実装するが、MVPを見送るかどうかの最終判断が必要。

4. **相談データ (JSON) の保存先**
   - 状態: 未決
   - 詳細: 取扱い対象となる構造化データ（JSON）をデータベース等に保存するかどうか。揮発させてAPIレスポンスの形だけで済ますか。ArchitectのDB選定判断待ち。

5. **`consultation_id` の生成責任**
   - 状態: 未決
   - 詳細: フロントエンド側でUUIDを生成してAPIに渡すか、バックエンド側で採番して返すか。API設計時に決定する。

6. **マスキングの具体方式**
   - 状態: 未決
   - 詳細: LLM送信前の簡易マスキング処理について、正規表現ベースでどの範囲（電話番号、メアド、固定の辞書語など）まで実装するか。MVPの運用の中で調整していく必要があるため実装上で現実解を探る。

7. **画像の一時保存期間（TTL設定等）**
   - 状態: 未決
   - 詳細: 画像を短期間だけクラウドストレージ等に置く場合、自動削除の期間（例：1時間、1日など）をどう設定するか。あるいは完全オンメモリで処理しきれるか、Architectによるインフラの選定時に決める。

8. **外部AIプロバイダへの送信ポリシー**
   - 状態: 未決
   - 詳細: 具体的にOpenAI APIやAnthropicなどのAPIを使う場合において、「データ保持オプトアウト機能」などの設定を確実に適用する仕組みの実装について検討要。

9. **本番前に必要な最小限のプライバシー文言（法務）**
   - 状態: 未決
   - 詳細: 画面注意文だけでなく、ごく簡潔な「利用規約」や「プライバシー方針」といった単独ページのリンクが必要になる可能性がある。法的な運用に耐えうるか、MVPテストユーザー次第で後回しかを今後決定する。
