# 養豚現場相談AI MVP 未決事項一覧 (Decision Log)

現在MVP仕様を策定した段階で、将来の拡張や実装時に決定すべき未決事項を以下にまとめる。

1. **写真付き相談時の具体的なプロンプト・インフラの扱い**
   - 状態: 未決
   - 詳細: MVPにおいて、画像つきリクエストをLLM（例えばGPT-4Vなど）に直接投げるのか、一旦画像URL化してバックエンドで処理するのか、インフラの具体的な接続方法が決まっていない。まずはダミーAPIまたはテキスト同等のモックでフロント実装を進めるか、即座にVLMへ繋ぐか、今後のArchitectタスクにて最小構成を決定する。

2. **`image_card_id` を用いた定型画像カードの表示の仕組み**
   - 状態: 未決
   - 詳細: 将来的に「下痢が増えたときにまず見る3点」などのクラウド上の画像リソースをフロントエンドで引っ張る仕組み。MVPではフロントエンド側は `image_card_id` を受け取るが無視する（あるいはダミー画像を出す）だけの対応で良いか、POと合意が必要（本仕様書ではMVP対象外としてダミーを許容している）。

3. **「獣医共有用メモを作る」ボタンの実装有無**
   - 状態: 未決
   - 詳細: ユーザー要望（要求仕様記載のUI要件）にあるが、必須事項ではない。クリップボードコピー機能を実装する工数がMVPスコープに収まるのであれば実装するが、見送るかどうかの最終判断が必要。

4. **相談データ (JSON) の保存先**
   - 状態: 未決
   - 詳細: ログをそのまま出力するだけでなくDB（Firestore, Supabase, RDBなど）に保存するかどうか。一時的なログファイルレベルに留めるのか。MVPのバックエンド実装における Architect のDB選定判断待ち。

5. **`consultation_id` の生成責任**
   - 状態: 未決
   - 詳細: フロントエンド側でUUIDを生成してAPIに渡すか、バックエンド側で採番して返すか。設計依存のためAPI設計時に決定する。
